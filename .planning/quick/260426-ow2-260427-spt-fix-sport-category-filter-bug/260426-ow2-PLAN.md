<objective>
Fix the sport category filter bug. User reported (feedback row 2ddac7cc-778a-44be-b4ea-8c792c79f01c, 2026-04-20) that picking Sports returned non-sport questions.

Root cause hypothesis: the RPCs `random_published_questions_excluding` and `count_available_questions` filter via the legacy `questions.category_id` column. The pipeline now writes multi-category data into `question_categories` (Phase 999.8). Questions whose primary `category_id` was misclassified by the Category Agent will leak into Sports if the join data disagrees, or vice-versa. With ~600 questions still not backfilled into `question_categories`, we cannot just switch to the join — we need a compatibility shim.

Approach: rewrite both RPCs so a question matches a category filter if EITHER (a) a row exists in `question_categories` linking it to any category in the requested subtree, OR (b) no `question_categories` rows exist for that question yet AND its legacy `category_id` is in the subtree. This honors the new model where it's authoritative and falls back to legacy data only for un-backfilled rows.

Purpose: stop returning wrong-category questions; preserve pool size during the backfill window.
Output: migration 00025 + reproduction verification.
</objective>
