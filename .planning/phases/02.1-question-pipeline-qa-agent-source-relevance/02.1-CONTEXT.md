**Gathered:** 2026-04-05

<domain>
## Phase Boundary

Add a QA Agent as the 5th pipeline step that reviews questions for natural language quality, category fit, difficulty calibration, and distractor quality -- rewriting fixable issues and rejecting broken questions. Move auto-publish responsibility from Fact-Check Agent to QA Agent. Improve Knowledge Agent with better Wikipedia search queries (category + subcategory terms), Haiku-powered source relevance filtering, and fallback search terms when initial queries return poor results.

</domain>

<decisions>
## Implementation Decisions

### QA Agent Scope & Checks
- **D-01:** QA Agent runs **4 quality checks** on each question: natural language quality, category fit, difficulty calibration, and distractor quality. Readability and question length are part of the natural language quality check -- no separate metric.
- **D-02:** QA Agent **rewrites fixable problems** (awkward phrasing, weak distractors, unclear wording) and passes them through. Only truly broken questions (wrong category, nonsensical, unanswerable) are rejected. Maximises yield from the pipeline.

### Auto-Publish Handoff
- **D-03:** Auto-publish moves from Fact-Check Agent to QA Agent. Publication requires **both** verification score >= 3 AND QA pass. Fact-Check Agent should be modified to remove its auto-publish logic -- it only sets verification scores and status (verified/rejected).
- **D-04:** QA Agent **reviews all verified questions** (score >= 1), not just score >= 3. Score 1-2 questions get quality-checked and stay 'verified' (not published). Score >= 3 questions get quality-checked and published if they pass all checks.
- **D-05:** When QA rewrites a question, it **keeps the existing verification score** -- no re-verification needed. QA only changes phrasing/distractors, not factual content, so the fact-check result still holds. No pipeline loops.

### Source Relevance Filtering
- **D-06:** Knowledge Agent uses **Claude Haiku to score source relevance** after fetching Wikipedia articles. Send title + first paragraph with category context to Haiku, get a relevance score (0-1). Only store sources above a threshold. Cheap (~$0.001 per check) and accurate.
- **D-07:** Knowledge Agent builds **richer search queries** using category name plus parent/child context from the category hierarchy in the DB. E.g., for 'Quidditch' search 'Quidditch Harry Potter sport rules' not just 'Quidditch'.
- **D-08:** Knowledge Agent retries with **one fallback search** using broader terms when initial query returns < 2 results (e.g., drop most specific term, use parent category name). One retry only -- no infinite loops.

### Pipeline Ordering
- **D-09:** Pipeline sequence is: Category -> Knowledge -> Questions -> Fact-Check -> **QA** (5th step). QA runs last with full context -- verification scores are available for publish decisions.
- **D-10:** QA Agent processes all questions that Fact-Check touched (score >= 1). Consistent quality across the board.

### Claude's Discretion
- Haiku relevance score threshold (suggested 0.5-0.7 but planner can tune)
- Exact QA Agent prompt structure and scoring rubric
- How to structure the rewrite prompt vs the scoring prompt (single call or two)
- Batch sizes for QA processing
- Whether to track QA rewrites in a separate log/column for observability

</decisions>
