# Phase 999.8 Plan 03: Pipeline Agent Rewrites — Multi-Category Shape Summary

One-liner: Rewrote Questions Agent and Calibrator Agent to emit/consume per-category percentage scores using a new join-table contract, backed by 5 new helper files and 17 passing Wave 0 unit tests.

## What Was Built

Three tasks, all complete:

**Task 1 — Helper files + extended schemas:**
- `pipeline/src/lib/slug-converter.ts`: `jsonKeyToSlug` / `slugToJsonKey` for snake_case↔kebab-case conversion
- `pipeline/src/lib/general-knowledge-guard.ts`: `GENERAL_KNOWLEDGE_SLUG` constant + `validateHasGeneralKnowledge`, `assertGeneralKnowledge`, `assertNoGeneralKnowledgeInExtras` guards
- `pipeline/src/lib/schemas.ts`: Replaced `difficulty` enum with `category_slugs + category_scores` in `QuestionGeneratedSchema`; added `CalibratorScoreSchema`
- `pipeline/src/lib/config.ts`: Added `DIFFICULTY_BANDS` constant (`hard 0-33, medium 34-66, easy 67-100`) + `scoreToBand` helper
- `pipeline/src/lib/categories.ts`: `resolveSlugsToIds` helper that queries categories table and throws on unknown slug

**Task 2 — Questions Agent rewrite:**
- Updated SYSTEM_PROMPT with multi-category JSON contract: `category_slugs` (1-3 extras, no GK), `category_scores` (snake_case keys including `general_knowledge`)
- Scoring framing added: integers 0-100 representing % of category players who'd answer correctly
- Pre-insert guards: `assertGeneralKnowledge` + `assertNoGeneralKnowledgeInExtras`; skip on failure
- `resolveSlugsToIds` called on `[...category_slugs, 'general-knowledge']`
- Single multi-row `question_categories` insert after question insert
- Transitional placeholders: `category_id = GK UUID`, `difficulty = 'normal'` until Plan 05 drops those columns

**Task 3 — Calibrator Agent rewrite:**
- Exported `calibrateQuestion(supabase, tokenAcc, input, claude, config)` with `CalibrateInput`/`CalibrateOutput` types
- New SYSTEM_PROMPT: "% of players who *chose to play this category* who'd get it right" (D-10 exact phrasing)
- Parses with `CalibratorScoreSchema`, validates all expected slug scores present
- Converts JSON keys to slugs via `jsonKeyToSlug`, resolves to UUIDs, upserts `question_categories`
- Removed all `calibration_percent` and `calibrated_at` writes (D-11)
- `runCalibratorAgent` batch orchestrator uses `calibrateQuestion` internally
