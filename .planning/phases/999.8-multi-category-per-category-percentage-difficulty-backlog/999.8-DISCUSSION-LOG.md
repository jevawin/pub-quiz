## Observed Score Refresh

### Refresh mechanism

| Option | Description | Selected |
|--------|-------------|----------|
| Nightly cron job | Aggregates question_plays once/day, upserts observed_score + observed_n | ✓ |
| Postgres trigger on question_plays insert | Real-time but adds write latency, overkill before n=30 | |

**User's choice:** Nightly cron job.

---

### Score used for quiz selection

| Option | Description | Selected |
|--------|-------------|----------|
| estimate_score until observed_n ≥ 30, then observed_score | CASE WHEN observed_n >= 30 THEN observed_score ELSE estimate_score END | ✓ |
| Always estimate_score | Crowd data never improves selection, contradicts system goal | |
| Always observed_score, fallback to estimate if null | Noisy at low n (3/5 = 60% is statistically unreliable) | |

**User's choice:** estimate_score until n ≥ 30, then observed_score. Threshold confirmed at 30 (no Bayesian blend).
