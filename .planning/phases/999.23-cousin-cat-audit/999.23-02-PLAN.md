    Create pipeline/src/scripts/seed-mistag-worklist.ts that:

    1. Loads env via dotenv override + creates service-role supabase client.

    2. Runs four heuristic queries against `question_categories` joined to `questions` and `categories`. Approach: for each target slug (`90s-music-hits`, `classic-sitcoms`, `traditional-pub-games`, `world-cuisine`), query all Qs currently tagged with that slug, then filter in-process per the heuristic rule:
       - 90s-music-hits: regex match in question_text against `/aphex twin|madeon|alunageorge|monstercat|sukiyaki|skrillex|deadmau5|odesza/i` → flag
       - classic-sitcoms: regex match against `/black mirror|rick and morty|m\*?a\*?s\*?h|inspector morse|breaking bad|game of thrones/i` → flag
       - traditional-pub-games: regex match against `/bowling|monopoly|video game|playstation|xbox|nintendo/i` → flag
       - world-cuisine: regex match in question_text against food allow-list `/food|dish|cuisine|recipe|drink|beverage|cocktail|wine|beer|cheese|bread|meat|fish|sauce|soup|dessert|pastry|fruit|vegetable|spice|coffee|tea/i` → if NO match, flag

    3. For each flagged Q, fetch all current `question_categories` rows (`slug`, `estimate_score`) via a join query — populate `current_slugs[]`.

    4. Resolve the 7 short-prefix Q ids from PROGRESS.md into full UUIDs by querying `questions.id LIKE '<prefix>%'` for each prefix; if multiple matches, log a warning and pick the one whose tags overlap the PROGRESS.md context. Add each as a `progress-manual` entry with `flag_reason` taken verbatim from PROGRESS.md context where possible.

    5. Merge all entries (heuristic + manual), de-duplicating by `question_id` (manual entries win on collision — they have richer flag_reason). Set `decision: null` on every entry.

    6. Write the merged array to `.planning/phases/999.23-cousin-cat-audit/data/mistag-worklist.json` with `JSON.stringify(entries, null, 2)`.

    7. Print summary to stdout: `{ heuristic_90s, heuristic_sitcoms, heuristic_pubgames, heuristic_cuisine, manual: 7, total_unique }`.

    Then run the script: `npx tsx pipeline/src/scripts/seed-mistag-worklist.ts`. Confirm output file ≥30 entries.

    If heuristic queries surface fewer than 23 entries (so total < 30 with the 7 manual), broaden the regex allow-lists incrementally OR add commentary to SUMMARY noting actual surfaced count — but do NOT pad the worklist with arbitrary entries.
