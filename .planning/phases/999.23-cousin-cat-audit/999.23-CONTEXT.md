## Decisions (HOW)

### D1 — Mistag worklist seeding

Hybrid: heuristic SQL queries surface candidates against known anti-patterns from PROGRESS.md, plus manual entries for the 7 explicit Q ids in PROGRESS.md.

Heuristic queries (run once, results piped into `data/mistag-worklist.json`):
- Qs tagged `90s-music-hits` whose `question_text` mentions an artist not in a 90s allow-list (Aphex Twin, Madeon, AlunaGeorge, Monstercat, Sukiyaki, generic 2010s electronic).
- Qs tagged `classic-sitcoms` whose `question_text` mentions Black Mirror, Rick and Morty, M*A*S*H, Inspector Morse, etc.
- Qs tagged `traditional-pub-games` whose `question_text` mentions bowling, Monopoly, video games.
- Qs tagged `world-cuisine` whose `question_text` does NOT mention a food / dish / drink keyword.

Worklist schema (`data/mistag-worklist.json`):
```json
[
  {
    "question_id": "uuid",
    "question_text": "...",
    "current_slugs": [{"slug":"...","estimate_score":N}],
    "flag_reason": "tagged 90s-music-hits but Q is about Madeon (2010s)",
    "suggested_action": "remove 90s-music-hits, add electronic-music",
    "decision": null
  }
]
```

User fills `decision` ∈ `fix | keep | defer` during conversational batch review.

### D2 — Apply mechanism

New script `pipeline/src/scripts/apply-cousin-changes.ts`. Mirrors `apply-chain-rows.ts` shape (stdin JSON, service-role client, idempotent). Supports two op types:

```typescript
type ChangeOp =
  | { op: 'delete'; question_id: string; slug: string; reason: string }
  | { op: 'insert'; question_id: string; slug: string; estimate_score: number; cousin_reason?: string }
  | { op: 'set_primary'; question_id: string; new_category_slug: string; reason: string };
```

Per Q: deletes run BEFORE inserts to respect cap-5 trigger. Each op appends one line to `data/audit-changes.jsonl` before commit.

### D3 — Batch presentation format

Markdown table per batch, posted in chat:

| Q (≤80 chars) | Correct | Current slugs (score) | Proposed change | Reason |
|---|---|---|---|---|
| ... | ... | `a` 75 / `b` 60 | DEL `b`, INS `c` (score 70) | tagged b but Q is about c |

Batch sizes:
- Cap-5 batch: 10 Qs per batch (19 total → 2 batches).
- Mistag batches: 15 Qs per batch (≈30 → 2 batches; expandable as heuristic queries surface more).

User responds row-by-row or with bulk "all rows OK" / "rows 1,3,5 only". Apply on confirmation.

### D4 — Cap-5 prune strategy

Default heuristic for the 19 cap-5 Qs:
1. Compute the to-delete candidate set: existing slugs that are NEITHER in the chain ancestors needed (per `batch-001.json`) NOR `general-knowledge`.
2. Drop the lowest-score candidate first.
3. If still over cap after inserts, drop next-lowest.
4. NEVER drop a chain ancestor (chain rows immutable per 999.22).
5. Drop GK only if its `estimate_score < 20` (matches 999.22 GK_THRESHOLD); otherwise keep.

User confirms each drop (judgment call when scores tie).

### D5 — Audit log schema

`data/audit-changes.jsonl` — append-only JSONL, one line per DB write:

```json
{"ts":"2026-05-09T22:40:00Z","batch_id":"cap5-001","question_id":"6a535027-...","op":"delete","slug":"british-history","prev_score":60,"reason":"low-score cousin pruned to free cap-5 for chain ancestors"}
{"ts":"2026-05-09T22:40:01Z","batch_id":"cap5-001","question_id":"6a535027-...","op":"insert","slug":"history","new_score":80,"cousin_reason":null,"chain_ancestor":true}
```

`cousin_reason` MUST be present and non-empty for inserts of slugs NOT in the chain ancestors. `chain_ancestor: true` flag distinguishes 999.22 ancestor inserts from new cousin tags.

### D6 — `pop-culture` cousin policy

Add `pop-culture` cousin only when the Q's primary leaf category is one of:
- Marvel / DC / comic franchises → `pop-culture` (alongside chain).
- Disney film franchises (Aladdin, Frozen, Lion King, etc.) → `pop-culture`.
- Star Wars → `pop-culture`.
- Mainstream gaming franchises (Mario, Zelda, Pokemon, Call of Duty, Fortnite, GTA) → `pop-culture`.

Do NOT auto-add to all entertainment Qs (e.g. obscure indie films, classical music, niche TV). Strict rule from ROADMAP §999.23: "a player picking that cat would genuinely expect this Q."

Score for new `pop-culture` cousin: default 60 (broad audience would recognise mainstream pop-culture refs); user may adjust per Q.

### D7 — Primary category moves

Use `set_primary` op when `questions.category_id` points to a wrong leaf (e.g. Aladdin Q at `literature` leaf because source was a book). Move to the right leaf, then chain ancestors are already in `question_categories` from 999.22 (no further action needed for chain).
