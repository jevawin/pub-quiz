<interfaces>
Per-batch flow:
1. Load mistag-worklist.json. Filter entries with `decision: null`. Take first ~15.
2. Present markdown table per D3:

| # | Q (≤80 chars) | Current slugs (score) | Flag reason | Suggested action | Decision (f/k/d) |
|---|---|---|---|---|---|

3. User responds row-by-row (`1: fix`, `2: keep`, `3: defer, ambiguous`, etc.) OR bulk (`all fix`, `1-10 fix, 11-15 defer`).

4. Executor constructs ChangeOps for `fix` rows:
   - Removing a wrong tag → `delete` op
   - Adding correct cousin → `insert` op with `cousin_reason` (non-empty per D5)
   - Wrong primary category → `set_primary` op
   - Per D7: when fixing primary, chain ancestors are already in question_categories from 999.22 — no new chain inserts needed

5. Apply: `cat batch.json | npx tsx pipeline/src/scripts/apply-cousin-changes.ts`.

6. Update mistag-worklist.json: set `decision` field on every entry in this batch (fix / keep / defer per user response).

7. Commit: `git add data/audit-changes.jsonl mistag-worklist.json mistag-batch-NNN.json && git commit -m "phase(999.23): mistag-NNN — N fix / M keep / K defer"`.

Pop-culture cousin policy reminder (D6): only add `pop-culture` for MCU / Disney / Star Wars / mainstream gaming franchises. Default score 60. NEVER add `pop-culture` to obscure indie films, classical music, niche TV.
</interfaces>
