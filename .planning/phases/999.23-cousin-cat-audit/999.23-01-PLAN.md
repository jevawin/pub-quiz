<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin → script | Untrusted JSON input crosses into service-role write context |
| script → Supabase service-role | Bypasses RLS; any malformed op could mutate question_categories beyond intended scope |
| script → audit log fs append | Local-only write; failure to append must NOT silently succeed the DB write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99923-01 | Tampering | stdin payload | mitigate | Strict shape validation per op type before any DB call; exit 1 on shape violation |
| T-99923-02 | Elevation of Privilege | service-role key in env | accept | Service-role required for the audit; key stays in env, never logged. Mirrors apply-chain-rows.ts posture |
| T-99923-03 | Repudiation | DB writes without audit trail | mitigate | Synchronous appendFileSync after each successful op; failure to write log surfaces as warning but DB write already committed (acceptable per D5 — JSONL is best-effort but per-op) |
| T-99923-04 | Denial of Service | unbounded ops array | accept | Operator-driven script; batch sizes bounded by D3 (10-15 Qs per batch) |
| T-99923-05 | Information Disclosure | audit log contains question_text? | mitigate | Audit log records IDs + slugs + reasons only; never embeds question_text |
</threat_model>
