# Plan 02: Supabase Migrations Summary

**Two migrations add the database surface for web quiz play tracking and question fetching — insert-only RLS locks down writes to the session owner, and a recursive RPC returns random published questions from any category subtree.**

## What Was Built

1. **Migration 00005** — `question_plays` and `quiz_sessions` tables with insert-only RLS. Both use `session_id = auth.uid()` to scope writes. No read/update/delete policies.

2. **Migration 00006** — `random_published_questions(p_difficulty, p_category_slug, p_limit)` RPC. Uses `WITH RECURSIVE cat_tree` to walk the category hierarchy, then returns N random published questions matching the difficulty and category subtree.

3. **Anonymous auth** — Enabled in the Supabase dashboard (manual step).
