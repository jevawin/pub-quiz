**pipeline/src/run-pipeline.ts**: Create the sequential pipeline orchestrator per D-05 and D-07.

Export `runPipeline()` as the main function. Also make it self-executing when run directly (top-level await or main guard).

Use `log()` from `./lib/logger.ts` instead of raw `console.log`/`console.error` for ALL output. This makes GitHub Actions log output much easier to parse and debug.

Logic:
1. Load config via `loadConfig()` (reads env vars, loads dotenv in non-CI environments)
2. Create Supabase client via `createSupabaseClient(config.supabaseUrl, config.supabaseServiceRoleKey)`
3. **Concurrent run guard:** Before creating a new pipeline_runs record, check for any existing record with `status = 'running'`:
   ```typescript
   const { data: running } = await supabase
     .from('pipeline_runs')
     .select('id, started_at')
     .eq('status', 'running')
     .limit(1);
   
   if (running && running.length > 0) {
     log('warn', 'Another pipeline run is already in progress, skipping this run', {
       existing_run_id: running[0].id,
       started_at: running[0].started_at,
     });
     process.exit(0); // Not an error -- just a skip. Cron + manual dispatch could overlap.
   }
   ```
   This is a simple mutex to prevent concurrent pipeline runs (cron + manual dispatch overlap). It exits with code 0 (not 1) because a skip is not a failure -- GitHub Actions will mark the step as success.
4. Create a shared `TokenAccumulator` via `createTokenAccumulator()` -- shared across all agents for cumulative budget tracking
5. Create pipeline_runs record:
   ```typescript
   const { data: run } = await supabase
     .from('pipeline_runs')
     .insert({
       status: 'running',
       config: {
         categoryBatchSize: config.categoryBatchSize,
         knowledgeBatchSize: config.knowledgeBatchSize,
         questionsBatchSize: config.questionsBatchSize,
         claudeModelGeneration: config.claudeModelGeneration,
         claudeModelVerification: config.claudeModelVerification,
         budgetCapUsd: config.budgetCapUsd,
       }
     })
     .select('id')
     .single();
   ```
6. Run agents sequentially in try/catch (per D-05 and D-07):
   ```typescript
   try {
     log('info', 'Starting Category Agent');
     const categoryResult = await runCategoryAgent(config, tokenAccumulator);
     log('info', 'Category Agent complete', { processed: categoryResult.processed, failed: categoryResult.failed });
     
     log('info', 'Starting Knowledge Agent');
     const knowledgeResult = await runKnowledgeAgent(config);
     log('info', 'Knowledge Agent complete', { processed: knowledgeResult.processed, failed: knowledgeResult.failed });
     
     log('info', 'Starting Questions Agent');
     const questionsResult = await runQuestionsAgent(config, tokenAccumulator);
     log('info', 'Questions Agent complete', { processed: questionsResult.processed, failed: questionsResult.failed });
     
     log('info', 'Starting Fact-Check Agent');
     const factCheckResult = await runFactCheckAgent(config, tokenAccumulator);
     log('info', 'Fact-Check Agent complete', { processed: factCheckResult.processed, failed: factCheckResult.failed });
     
     // Mark success
     await supabase.from('pipeline_runs').update({
       status: 'success',
       completed_at: new Date().toISOString(),
       categories_processed: categoryResult.processed,
       categories_failed: categoryResult.failed,
       sources_fetched: knowledgeResult.processed,
       sources_failed: knowledgeResult.failed,
       questions_generated: questionsResult.processed,
       questions_failed: questionsResult.failed,
       questions_verified: factCheckResult.processed,
       questions_rejected: factCheckResult.failed,
       total_input_tokens: tokenAccumulator.input_tokens,
       total_output_tokens: tokenAccumulator.output_tokens,
       estimated_cost_usd: tokenAccumulator.estimated_cost_usd,
     }).eq('id', run.id);
     
     log('info', 'Pipeline complete', { cost_usd: tokenAccumulator.estimated_cost_usd.toFixed(4) });
   } catch (error) {
     // Per D-07: stop and report, partial work is kept
     await supabase.from('pipeline_runs').update({
       status: 'failed',
       completed_at: new Date().toISOString(),
       error_message: error instanceof Error ? error.message : String(error),
       total_input_tokens: tokenAccumulator.input_tokens,
       total_output_tokens: tokenAccumulator.output_tokens,
       estimated_cost_usd: tokenAccumulator.estimated_cost_usd,
     }).eq('id', run.id);
     
     log('error', 'Pipeline failed', { error: error instanceof Error ? error.message : String(error) });
     process.exit(1); // GitHub Actions marks step as failed
   }
   ```

7. Add a self-executing guard at the bottom:
   ```typescript
   // Run when executed directly
   runPipeline();
   ```

**`.github/workflows/question-pipeline.yml`**: Create the GitHub Actions workflow per D-01 and PIPE-09.

```yaml
name: Question Pipeline

on:
  schedule:
    - cron: '23 4 * * *'    # Daily at 04:23 UTC (odd time avoids GH Actions congestion per Pitfall 6)
  workflow_dispatch:          # Allow manual trigger from GitHub UI
    inputs:
      budget_usd:
        description: 'Budget cap in USD for this run'
        required: false
        default: '1.00'
      category_batch_size:
        description: 'Number of new categories to propose'
        required: false
        default: '5'
      knowledge_batch_size:
        description: 'Number of categories to fetch Wikipedia sources for'
        required: false
        default: '10'
      questions_batch_size:
        description: 'Number of questions to generate'
        required: false
        default: '20'

jobs:
  run-pipeline:
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: pipeline/package-lock.json

      - name: Install dependencies
        working-directory: pipeline
        run: npm ci

      - name: Run pipeline
        working-directory: pipeline
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
          PIPELINE_BUDGET_USD: ${{ github.event.inputs.budget_usd || '1.00' }}
          CATEGORY_BATCH_SIZE: ${{ github.event.inputs.category_batch_size || '5' }}
          KNOWLEDGE_BATCH_SIZE: ${{ github.event.inputs.knowledge_batch_size || '10' }}
          QUESTIONS_BATCH_SIZE: ${{ github.event.inputs.questions_batch_size || '20' }}
        run: npx tsx src/run-pipeline.ts
```

This workflow:
- Runs daily at 04:23 UTC (odd time to avoid GitHub Actions congestion per Pitfall 6)
- Supports manual trigger with configurable batch sizes and budget (workflow_dispatch with inputs)
- Uses Node.js 22 with npm cache for fast installs
- Uses `npm ci` which requires the `pipeline/package-lock.json` committed in Plan 01-01
- Passes all config as environment variables (secrets for keys, inputs for tunables)
- 30-minute timeout prevents runaway runs
- Pipeline's process.exit(1) on failure makes the GitHub Actions step fail, triggering default notification
- The concurrent run guard in the orchestrator (Plan 01-04 Task 1) handles overlap between cron and manual dispatch -- if both trigger simultaneously, the second one exits 0 (skip)
- Per D-07: no retry logic -- if it fails, the step fails and GitHub notifies the repo owner
