<objective>
Build the Category Agent and Knowledge Agent -- the first two stages of the sequential pipeline.

Purpose: The Category Agent discovers subcategories under the 12 seed themes, growing the category tree. The Knowledge Agent fetches Wikipedia reference material for categories that lack sources, creating the knowledge base that the Questions Agent and Fact-Check Agent will use. These two agents are the "input" side of the pipeline.

Output: Two working agent scripts with unit tests, ready to be called by the pipeline orchestrator.
</objective>

**pipeline/src/agents/category.ts**: Create the Category Agent per D-08 (separate script) and PIPE-04.

Export `runCategoryAgent(config: PipelineConfig, tokenAccumulator: TokenAccumulator): Promise<AgentResult>`.

Define `AgentResult` interface: `{ processed: number, failed: number }`.

Logic:
1. Create Supabase client via `createSupabaseClient(config.supabaseUrl, config.supabaseServiceRoleKey)`
2. Create Claude client via `createClaudeClient(config.anthropicApiKey)`
3. Fetch all existing categories from Supabase: `select('id, name, slug, parent_id, depth')`
4. Build a text representation of the category tree for the Claude prompt context
5. Call Claude with model `config.claudeModelGeneration` (Sonnet):
   - System prompt: "You are a pub quiz category expert. Your job is to propose new subcategories for a quiz app. Categories should be specific enough to generate interesting questions but broad enough to have 20+ potential questions. Avoid overly niche topics."
   - User prompt: Include the existing category tree, ask for `config.categoryBatchSize` new subcategories, specify they must be children of existing categories, max depth 3
   - Request structured output matching `CategoryBatchSchema`
6. Parse response with `CategoryBatchSchema.parse()`
7. Track tokens: `trackUsage(response, tokenAccumulator, SONNET_INPUT, SONNET_OUTPUT)`
8. Check budget: `checkBudget(tokenAccumulator, config.budgetCapUsd)`
9. For each proposed category:
   - Look up `parent_slug` in existing categories to get parent_id and parent_depth
   - Skip if slug already exists
   - Skip if parent_depth + 1 > 3
   - Insert into `categories` table with: name, slug, parent_id, depth (parent_depth + 1), description, created_by='pipeline'
10. Return `{ processed: successCount, failed: failCount }`

**pipeline/src/agents/knowledge.ts**: Create the Knowledge Agent per D-08 and PIPE-05.

Export `runKnowledgeAgent(config: PipelineConfig): Promise<AgentResult>`.

Logic:
1. Create Supabase client
2. Find categories needing sources: query categories and left join sources, select categories with source count < 3, limit to `config.knowledgeBatchSize`
   - SQL approach: select categories, then for each check source count. Simpler: query all categories, then query sources grouped by category_id, find categories with count < 3
3. For each category needing sources:
   - Call `searchArticles(category.name, config.wikipediaUserAgent, 5)` to find relevant Wikipedia articles
   - For each article title returned:
     - Call `getArticleText(title, config.wikipediaUserAgent, config.wikipediaMaxContentLength)`
     - Skip if null (missing page)
     - Compute SHA-256 hash of the content using Node.js `crypto.createHash('sha256')`
     - Check if content_hash already exists in sources table (upsert or check-then-insert)
     - If new: insert into sources table with:
       - `category_id`: the category's UUID
       - `title`: the Wikipedia article title
       - `url`: `https://en.wikipedia.org/wiki/${encodeURIComponent(title.replace(/ /g, '_'))}`
       - `content`: the truncated plain text
       - `content_hash`: the SHA-256 hex string
4. Track: no Claude API calls in this agent (Wikipedia only), so no token tracking needed
5. Return `{ processed: sourcesInserted, failed: sourcesFailed }`
