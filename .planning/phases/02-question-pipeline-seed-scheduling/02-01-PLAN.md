    **seed-threshold-check.ts** (per D-03, D-11, D-12):

    Create `pipeline/src/seed-threshold-check.ts` with an exported async function `checkThreshold(supabase: TypedSupabaseClient)` that:
    1. Queries `questions` table with `select('*', { count: 'exact', head: true }).gte('verification_score', 3)` to get verified count
    2. Queries `questions` table with `select('category_id').gte('verification_score', 3)` then uses a Set to count unique categories
    3. Returns `{ seedComplete: boolean, verifiedCount: number, categoryCount: number }`
    4. If `seedComplete === true`:
       - Writes `seed_complete=true\n` and `verified_count=${count}\n` to `process.env.GITHUB_OUTPUT` file (using `fs.appendFileSync`)
       - Writes `::notice::SEED COMPLETE: ${verifiedCount} verified questions across ${categoryCount} categories` to stdout via `console.log`
    5. If `seedComplete === false`:
       - Logs progress: `Seed in progress: ${verifiedCount}/1000 verified questions across ${categoryCount} categories`
    6. The threshold constant is `const VERIFIED_THRESHOLD = 1000`
    7. On Supabase error, log the error and call `process.exit(1)`

    Add a self-execution guard at the bottom (same pattern as run-pipeline.ts):
    ```typescript
    const isDirectExecution = typeof process !== 'undefined' && process.argv[1] &&
      (process.argv[1].endsWith('seed-threshold-check.ts') || process.argv[1].endsWith('seed-threshold-check.js'));
    if (isDirectExecution) {
      const config = loadConfig();
      const supabase = createSupabaseClient(config.supabaseUrl, config.supabaseServiceRoleKey);
      checkThreshold(supabase);
    }
    ```

    **category-selection.ts** (per D-05):

    Create `pipeline/src/lib/category-selection.ts` with an exported async function:
    ```typescript
    export async function getEligibleCategoriesOrdered(
      supabase: TypedSupabaseClient,
      batchSize: number,
      minQuestionsThreshold?: number,
    ): Promise<Array<{ id: string; name: string; slug: string; questionCount: number }>>
    ```

    Implementation:
    1. Fetch all categories: `supabase.from('categories').select('id, name, slug')`
    2. For each category, count questions: `supabase.from('questions').select('*', { count: 'exact', head: true }).eq('category_id', cat.id)`
    3. Check sources exist: `supabase.from('sources').select('id', { count: 'exact', head: true }).eq('category_id', cat.id)`
    4. Filter: exclude categories with 0 sources OR with questionCount >= (minQuestionsThreshold ?? 10)
    5. Sort by questionCount ascending (least-covered first)
    6. Return first `batchSize` entries
    7. Import `log` from `../lib/logger.js` and log the selection: `log('info', 'Category selection', { eligible: N, selected: N, leastCovered: firstCat.name })`

    The `minQuestionsThreshold` defaults to 10 (matching the existing `MIN_QUESTIONS_THRESHOLD` in questions.ts).

    **Tests** -- follow the mocking pattern from existing tests (read `questions.test.ts` and `wikipedia.test.ts` first for the project's mock conventions). Use `vi.mock` for Supabase client. Create mock data that exercises the ordering and filtering logic.
